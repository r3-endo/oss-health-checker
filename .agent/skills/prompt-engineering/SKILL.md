# Prompt Engineering

## 使う場面
- Claude Code用のコマンド/フック/スキルを作成するとき
- サブエージェントやLLM向けのプロンプトを設計するとき
- 既存プロンプトの最適化・改善を行うとき

## 進め方（簡易）
1. 目的と出力条件を一つずつ確認する
2. 2〜3の設計方針を提示する
3. 最終案と次アクションを簡潔にまとめる

## 出力フォーマット（簡易）
- 目的/前提
- 主要な論点
- 代替案（2〜3）
- 結論
- 次アクション
