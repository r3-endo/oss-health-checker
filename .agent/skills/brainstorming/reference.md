# ブレインストーミング詳細方針

## 原則
- 一度に一つの質問だけを投げる（複数質問は避ける）
- 可能な限り選択式の質問を優先する
- YAGNIを徹底し、不要な機能を増やさない
- 2〜3のアプローチを提示してから結論を出す

## 質問設計のガイド
- 目的確認: 「何を達成したいか」「誰の課題か」を最初に押さえる
- 制約確認: 期限、予算、既存システムなどの制約を整理する
- 成功条件: 何が成功なのかを一文で定義する
- 優先順位: 重要度の高い論点から順に扱う

## 代替案の出し方
- 観点をずらして2〜3案を提示する（短期/中期、シンプル/拡張など）
- 各案は「メリット」「デメリット」「適用条件」を1行で添える
- 判断に必要な追加情報があれば次の質問に回す

## 結論のまとめ方
- 選ぶ理由を1〜2行で明示する
- リスクと前提条件を短く添える
- 次アクションを具体化する（誰が/何を/いつまでに）

## 使い分け（将来拡張用）
- 詳細な要件化が必要なら、要件整理のテンプレートを追加する
- 技術選定に寄せる場合は、評価軸の固定化を検討する
