# Phase 2: Registry Adoption Dashboard 仕様

## 1. 背景（Why）

現在の OSS Maintenance Health は GitHub の開発シグナル（Dev Health）を中心に可視化している。

しかし、ライブラリ選定において CTO / シニアエンジニアが重視するのは、

- 開発が回っているか（Dev Health）
- 実際に使われ続けているか（Adoption）
- 運用リスクは許容範囲か（Security / Governance）

の 3 層である。

Phase 2 では、GitHubとは別レイヤとして **Registry（採用実態）可視化機能** を追加する。

本フェーズではまず UI 設計と API 契約定義を行い、実データ取得は最小構成から開始する。

---

## 2. 目的（Objective）

- GitHub の Dev Health とは独立した Adoption レイヤを追加する
- npm を最初の対象レジストリとする
- 加工やスコアリングは行わず、一次情報を表示する
- 将来的に Maven / PyPI / crates.io / Homebrew へ拡張可能な設計にする

## 3. ecosystem-adoption レイヤで表示する項目（npm 初期版）

### 表示カラム

- Repository（owner/name）
- Package Name
- Weekly Downloads
- Downloads Δ 7d
- Downloads Δ 30d
- Last Published Date
- Latest Version

### 表示方針

- スコアは表示しない
- 生データに近い値を表示する
- Δ（増減）は数値のみ（矢印アイコンは任意）
- 未対応パッケージは "Not Mapped" 表示
